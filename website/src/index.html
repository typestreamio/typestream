Three steps. No infrastructure to learn.

1

Connect your database

Point TypeStream at your existing Postgres or MySQL. It reads the change log — no triggers, no polling, no app code changes. Your application keeps writing to the database exactly like it does today.

2

Define your pipeline

Write a short config file that says where data goes and how to transform it. Or pipe commands together in the CLI — it reads like Unix pipes.

cat /dev/kafka/local/topics/products | text-extract | elasticsearch

Or as a config file — version-controlled, CI/CD friendly, reviewable in a PR.

3

Preview and deploy

Run typestream plan to see exactly what will change — like terraform for data pipelines. Review the diff, then apply.

$ typestream plan search-sync.json search-sync: + CREATE pipeline "search-sync" source: postgres://mydb/products steps: text-extract → elasticsearch 1 pipeline to create, 0 to update, 0 to delete. Run `typestream apply` to execute.

Common patterns

Deploy in minutes, not quarters.

Every developer
Keep search results fresh

A product changes in your database. Your search results still show the old name, the old price, the old description — until the next reindex runs.

TypeStream keeps Elasticsearch in sync with your database in real time. Every change is indexed within seconds. No cron job, no reindex script, no stale results.

Stop sensitive data from leaking

Customer emails, credit card numbers, and SSNs flow through your pipeline into analytics warehouses where they shouldn't be. You find out during an audit.

TypeStream scans every record in flight, detects PII patterns, and either redacts or blocks before the data reaches your warehouse. Compliance built into the pipeline, not bolted on after.

Real-time dashboards, not nightly batches

Your team makes decisions on dashboards that are 24 hours stale. By the time you see a spike, it already happened yesterday.

TypeStream streams database changes directly into your analytics warehouse. ClickHouse, Snowflake, BigQuery — your dashboards update as fast as your data changes.

AI-ready stack
Add semantic search to any database

Your users type natural language queries, but your database only does exact match or LIKE '%keyword%'. Building semantic search means standing up a vector database, writing an embedding pipeline, and keeping it in sync.

TypeStream generates embeddings from your database content and pushes them to Weaviate or any vector store. As your data changes, the embeddings update automatically. Semantic search without the plumbing.

Classify and route with AI

Every support ticket needs to be categorized. Every document needs a sentiment score. You're either doing it manually or building a custom ML pipeline.

TypeStream runs every record through an LLM in flight. Categorize, score, extract, summarize — then route the results to your warehouse or downstream system. Define the prompt in your pipeline config.

Generate REST APIs from your data

You need a fast endpoint that serves the current state of something — live prices, user profiles, inventory counts. So you build an API, a database query, a cache layer, and a cache invalidation strategy.

TypeStream materializes a live view from your data stream and serves it as an endpoint. No separate API server, no cache, no database query. The endpoint updates as fast as your data changes.

Data infrastructure
Replicate to your data warehouse

Getting data from your production database into your warehouse means setting up an ETL tool, scheduling jobs, handling failures, and hoping nothing drifts.

TypeStream streams changes from your production database directly into your warehouse. Every insert, update, and delete arrives in seconds. No ETL tool, no batch schedule, no drift.

Sync databases without custom code

You need a subset of your Postgres data in another database — maybe MySQL for a legacy system, maybe a read replica with different columns. So you write a sync script and pray it doesn't break.

TypeStream replicates, filters, and transforms between databases in real time. Define what goes where in a config file. Change it when your needs change.

Enrich data as it flows

Your raw data is missing context. IP addresses without locations. Product IDs without names. Order records without customer details. You enrich in batch jobs that run overnight.

TypeStream enriches records in flight — geo-locate IPs, join with reference data, resolve IDs to names. Every record arrives at its destination already enriched. No batch job, no overnight wait.

Built for production, not just demos

Plan before you deploy

Every change goes through typestream plan first. See exactly what pipelines will be created, updated, or deleted — before anything runs. Like terraform for data pipelines. Review in CI, approve in a PR.

Backpressure and delivery guarantees

If Elasticsearch is slow or Redis is down, TypeStream queues and retries automatically. No data loss, no duplicate processing. Events are delivered exactly once.

Observability included

Metrics, logs, and traces out of the box. Know how many events per second each pipeline processes, where bottlenecks are, and when something fails — before your users notice.
